Forms in HTML

  • A form is used to collect information from the user.
  • Example: Login form, signup form, search bar, feedback form.
  • In HTML, forms are created using the <form> tag.

      Form Elements (Inputs inside form)

      • This is the most common form tag. It's a self-closing tag that creates an input field. Its behavior changes based on its type attribute.

    1. Text Box: Used to take single-line input.

      2. The Need for Meaning - The <label>

      • Suppose you visit a website and see a form that has only input fields, but no labels. You will get confused â€” should you enter your name, email, address, or something else?
      • To solve this, we need a descriptive piece of text. The correct HTML tag for this is the <label>. It is specially designed to be the title/description for a form field.
      • Now, you can easily understand that you have to enter your First Name in the input field.
      • If we only write the label and input separately (without using for and id), then they have no connection and behave like different elements.

      The Need for Connection - id and for

      • To connect them, we use id in the input field and for in the label.
      • The value of for in the label must be the same as the value of id in the input.
      • Now, when you click on the text First Name:, the cursor will automatically move to the input box. This makes the form easier to use and more accessible.

      The Need for Submission - The <form> and submit

      • Collecting user input is not enough â€” we also need a way to send (submit) that data to the server.
      • In HTML, this is done using the <form> tag. It works as a container that holds all input fields.
      • To actually send the data, we use a submit button. The simplest way is <input type="submit">.
      • We now have a complete visual form with two fields and a submit button. When you click the button, the page reloads, but the data doesn't go anywhere yet.

      The Need for Data Identification - The name Attribute

      • When the form is submitted, the browser needs to package the data to send to a server. How does it label the data?
      • If a user types "Arjun" in the first box, how does the server know that "Arjun" is the first name? The id attribute is only for use within the page; it is not sent to the server.
      • For this, we use the name attribute in every input field. It works as an identifier for the data.
      • The name attribute becomes the key and the user 's input becomes the value. Together, they are sent to the server as key=value.
      • Now we have a truly functional form, ready to send meaningful data. When submitted, the browser will create a package that looks like this:
        • firstName = (whatever the user typed)
        • lastName = (whatever the user typed)

      The Need for "Select One" - Radio Buttons

      • Sometimes in a form, we need the user to choose only one option from a list. Example: selecting gender (Male / Female).
      • For this, HTML provides the radio button input type.
      • All radio buttons that belong to the same group must share the same name attribute. This ensures only one option can be selected at a time.
          • The meaning of these attributes
        • type="radio": Creates the circular radio button.
        • name="shirtSize": This is the critical part. Because all three have the same name, the browser knows they are a single group and will only let you select one.
        • id="sizeS": Each input still needs a unique id so its specific label can connect to it.
        • value="small": This is the actual data that will be sent to the server if this option is selected. If the user clicks "Small", the form will send shirtSize=small. Without the value, the data would be meaningless.

      The Need for "Select Many" - Checkboxes

      • Sometimes in a form, the user should be able to select more than one option at the same time. Example: choosing hobbies or interests.
      • For this purpose, HTML provides the checkbox input type.
      • Each checkbox works independently, so users can tick none, one, or multiple options as per their choice.
          • The meaning of these attributes
        • type="checkbox": Creates the square checkbox.
        • name="toppings": All three share this name, telling the server they are all "toppings".
        • id="toppingPep": Each has a unique id for its label.
        • value="pepperoni": Each has a unique value to identify which topping was chosen.

      A Better Button - The <button> Element

      • In HTML, we can create buttons using <input type="submit"> or <input type="reset">. But these are limited â€” they cannot hold rich content like text + icons.
      • The <button> element is more powerful because it can contain text, images, or even HTML inside it.
          Types of <button>:
        • type="submit" â†’ submits the form data.
        • type="reset" â†’ resets/clears the form fields.
        • type="button" â†’ normal button (can be used with JavaScript).
          • The meaning of these attributes
          • <button>: The container for the button.
          • type="submit": This is very important. This attribute tells the button to act as a form submit button. (It can also be type="button" for JavaScript or type="reset").
          • <strong>Submit</strong>: We can now put other HTML tags, like <strong> or even an <img>, right inside our button!

      The Need for Long-Form Text - <textarea></textarea>

      • Normal <input type="text"> fields are designed for short answers like names, emails, or phone numbers.
      • But sometimes, users need to enter longer text, such as an address, feedback, or comments.
      • For this, HTML provides the <textarea> element.
        Features of <textarea>:
        • It can hold multiple lines of text.
        • You can set its rows and cols size.
        • Users can expand/shrink it (in most browsers).
      • rows="4": This attribute controls the visible height of the text area, suggesting it should be about 4 lines of text tall.
      • cols="50": This controls the visible width, suggesting it should be about 50 characters wide.

      The Need for Many Options - The Dropdown (<select>)

      • Sometimes we need users to choose one option from a large list. Example: selecting a country, course, or year of study.
      • Writing multiple radio buttons for this would be messy and space-consuming.
      • HTML provides the <select> element to create a dropdown menu.
      • Inside a <select>, we use <option> tags to define the available choices.
      • <select>: This is the main container for the dropdown. The id and name attributes go on this tag.
      • <option>: Each individual choice in the dropdown is an <option> tag.
      • value Attribute on <option>: This is the data that gets sent to the server. The text between the tags (India) is what the user sees.

      More Specialized Inputs (HTML5 Power-ups)

      • HTML5 introduced many new type attributes for <input> to make forms smarter and more user-friendly.

    2. Password Box: Hides the text when typing.


    3. Number Input with min and max: Accepts only numbers within the given range (18-60).


    4. Date Input: Shows a calendar date picker, making it easier and error-free to select dates.


    5. Color Input: Opens a color picker to choose a color.


    6. Range Input: Creates a slider control (useful for volume, brightness, etc.)


    7. File Input: Lets the user upload a file from their device.


    8. Required Attribute: Marks a field as mandatory (form won't submit if left empty).


    9. Placeholder Attribute: Shows a hint inside the input until the user types something.


    10. Telephone Input: Used to enter phone numbers. Some browsers show a number keypad on mobile for easier typing.
