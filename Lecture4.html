File Path system

  • A file path tells the browser where to find a file (like an image, CSS, JS, or another page).
  • It is used inside attributes such as src (for images, scripts) or href (for links, CSS).
  • File paths can be absolute (full location) or relative (based on current file location).

    There are two types of File Paths

    1. Relative File Paths:
      • It shows the location relative to the current HTML file.
      • You will use this 99% of the time for linking your own files together (images, CSS, other HTML pages).

    2. Absolute File Paths:
      • It shows the complete URL or location of the file. It starts with http:// or https://.
      • When to use it: You ONLY use absolute paths when you are linking to a resource that is NOT on your own website.
        • Linking to another website.
        • Using a font from Google Fonts.
        • Using an image from an image-hosting service.
    • Why We Don't Use Absolute File Paths -
    • Works only on your computer â†’ Path like C:/Users/... runs only locally, not on other systems.
    • Not portable â†’ If you move project to another folder/server, links will break.
    • Different OS problem â†’ Windows uses \ while Linux /, macOS use /, so paths mismatch.
    • Security issues â†’ Sharing absolute paths may expose your personal computer's directory structure (like username, system details), which is unsafe.
    • Bad practice â†’ In professional web development, only relative paths are recommended.

HTML Boilerplate Code

  • This is a block of code that you will start every single HTML file with. It's not something you need to reinvent each time; you just copy-paste it and fill in the blanks.
  • It's the "blueprint of the house" that you need before you can start putting in the walls (<h1>) and windows (<img>). It looks like this - below is an image of the boilerplate. ðŸ‘‡

The explanation of this BoilerPlate Code -

  • <!DOCTYPE html> tells the browser that the document is written in HTML5 (the 5th version of HTML).
  • <html lang="en"> ... </html>: This is the root element that wraps your entire page. The lang="en" attribute declares that the primary language of the page content is English. This is very important for accessibility and SEO. You'd change "en" to "es" for Spanish, "hi" for Hindi, etc.
  • <head> ... </head>: The <head> tag is used to contain information about the webpage (not shown directly on the page). It helps browsers, search engines, and devices understand the page better.
  • <meta charset="UTF-8"> ensures that all characters (letters, numbers, symbols, emojis, etc.) are displayed correctly.
  • <meta name="viewport" ... > helps the webpage look good and readable on mobiles, tablets, and desktops.
  • <title> ... </title>: The <title> tag sets the text that appears in the browser tab, in bookmark lists, and in search engine results. It's a critical piece of metadata for both usability and SEO.
  • <body> ... </body>: The <body> section is the container for everything the user will actually see on the page: your headings, paragraphs, images, links, tables, etc. All the tags you've learned so far go inside the <body>.

But websites work even without Boilerplate

  • Browsers are smart - They automatically fill missing parts like <html> or <head>.
  • Basic code is enough - Even if you write only text or a <p> tag, the browser will still show it.
  • Default settings - Browser uses its own defaults for things like font, size, or page width.
  • HTML is forgiving - It is made in such a way that even incomplete code can run.
  • But in real projects, boilerplate is important for correct display, responsiveness, SEO, and professional coding.

Multipage Website

  • A multipage website is a site made up of multiple linked HTML pages, each showing different information.
  • Each page has its own HTML file (like index.html, about.html, contact.html).
  • Pages are connected to each other using links (<a> tag).
  • Common example: Home, About Us, Services, Contact Us.
  • Used when website has a lot of information, which cannot fit into a single page.
